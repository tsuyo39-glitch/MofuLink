## 1. このアプリの立ち位置（市場リサーチの要約）

国内のペット記録アプリ（ペットノート+、うちの子手帳、ペトナビ 等）は機能が成熟しており、
「プロフィール・健康記録・日記・支出管理・共有」はどれも定番。
**定番機能は当然満たしたうえで、既存アプリの不満を解消する**ことが本アプリの存在価値。

| 既存アプリの不満 | 本アプリの対応 |
|---|---|
| 課金が高い・値上げされる | 課金なし |
| 広告が出る（課金しても出る） | 広告なし |
| 機種変更でデータが消えた | ローカル保存＋エクスポート＋クラウド同期 |
| 表示が遅い・固まる | 軽量実装、初期表示を最優先 |
| スマホ専用で作りが中途半端 | iPhone 専用に最適化（縦画面・片手操作前提で作り込む） |

### 最終目標（配布）

開発者本人だけでなく、**知り合い（家族・友人）も自分の iPhone でこのアプリを使えること**を最終目標とする。

- **配布方法:** GitHub（GitHub Pages）でホスティングし、HTTPS の URL を配る。各自が iPhone Safari で開き「ホーム画面に追加」して PWA として使う。
- **公開範囲:** 一般公開ではなく**知り合いのみ**。アクセス制限は「**見つけにくい URL を知り合いにだけ共有する**」方式とする（GitHub Pages の URL は技術的には誰でも開けるが、検索・宣伝はしない）。
- **データの独立性:** データは各端末のローカル（IndexedDB）に保存され、ユーザー間で共有・送信されない。したがって URL を共有しても**他人のデータが見えることはない**（各自が自分の記録を持つ）。
- **HTTPS の副次効果:** GitHub Pages は HTTPS 配信のため、PWA（Service Worker・オフライン・ホーム画面追加）が正しく動作し、`crypto.randomUUID` 等のセキュアコンテキスト依存 API も問題なく使える。

## 2. データモデル

### Pet（ペット）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意ID |
| name | string | 名前 |
| species | string | 種類（犬・猫・その他） |
| photoBlob | Blob? | プロフィール写真（圧縮済み） |
| birthday | string? | 生年月日（ISO） |
| adoptedDate | string? | 迎えた日（ISO） |
| favoriteFood | string? | 好きな食べ物 |
| memo | string? | 自由メモ |
| createdAt | string | 登録日時 |

- **現在の年齢は保存しない。birthday から `date-fns` で都度計算して表示する**（「◯歳◯ヶ月」）
- 複数ペット（多頭飼い）を登録可能。アプリ全体は「選択中のペット」を切り替えて使う

### DiaryEntry（日記）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意ID |
| petId | string | どのペットの記録か |
| date | string | 日付（ISO） |
| body | string | 本文 |
| photoBlobs | Blob[] | 写真（複数・圧縮済み） |
| createdAt | string | 作成日時 |

### Expense（支出）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意ID |
| petId | string | どのペットの支出か |
| date | string | 日付（ISO） |
| category | string | カテゴリ（下記） |
| amount | number | 金額（整数・円） |
| memo | string? | メモ（例: ◯◯動物病院） |
| createdAt | string | 作成日時 |

- **カテゴリ（初期値）:** 病院・餌・おやつ・トイレ用品・トリミング・保険・おもちゃ・その他
- カテゴリはユーザーが追加・編集できる

## 3. 画面・機能仕様

### 3-1. ペット プロフィール
- 一覧表示（カードで複数ペット、写真＋名前＋現在の年齢）
- 登録・編集フォーム: 写真、名前、種類、生年月日、迎えた日、好きな食べ物、メモ
- 写真はカメラ/アルバムから選択 →（保存前に圧縮）→ 保存
- 年齢は生年月日から自動計算して表示。「迎えてからの日数」も併せて表示

### 3-2. 日記（写真付き）
- 日付ごとの記録。1記録に複数枚の写真を添付可能
- **カレンダー表示**と**タイムライン表示**を切り替えられる（既存アプリで好評な機能）
- 記録のある日はカレンダー上で印を付ける
- 写真は一覧でサムネイル、タップで拡大

### 3-3. 支出管理
- 支出の登録（日付・カテゴリ・金額・メモ）
- 一覧（日付降順、カテゴリのアイコン/色分け）
- **集計機能（重要）:**
  - 月別合計
  - カテゴリ別の内訳（円グラフ＝Recharts）
  - 月ごとの推移（棒グラフ）
  - 期間指定での合計
- ペット別の集計（多頭飼い時に「この子にいくらかかったか」）

### 3-4. データ管理（差別化の核）
- **エクスポート:** 全データを JSON でダウンロード（手動バックアップ）
- **インポート:** JSON から復元
- Phase 2 ではここにクラウド同期のオン/オフが加わる

### Phase 1 — ローカルで完結する最小実用版（MVP）

- [x] リポジトリ層を抽象化（後でクラウドに差し替え可能に）
- [x] ペット プロフィール（登録・編集・一覧・年齢自動計算）
- [x] 写真の選択・圧縮・保存・表示
- [x] 日記（カレンダー＋タイムライン、写真複数）
- [x] 支出（登録・一覧・カテゴリ別集計グラフ・月別推移）
- [x] JSON エクスポート / インポート（手動バックアップ）
- [x] PWA 化（manifest・アイコン・オフライン・ホーム画面追加）
- [x] iPhone Safari で動作確認（ホーム画面追加・縦画面レイアウト・写真撮影）

### Phase 1.5 — GitHub Pages で配布（知り合いが使えるようにする）
ゴール: 知り合いが、配布された URL を iPhone Safari で開いてホーム画面に追加し、PWA として使い始められる状態。
Phase 1 の MVP が動いたら、Google Drive 連携（Phase 2）より先にここを行ってよい。

- [x] GitHub リポジトリを作成（公開リポジトリだが、URL は宣伝せず知り合いにのみ共有）
- [x] GitHub Pages のサブパス配信に合わせて Vite の `base` を設定（例: `/MofuLink/`）
- [x] PWA の `start_url` / `scope` / アイコンパスを `base` に合わせて調整（ホーム画面追加が正しく動くこと）
- [x] GitHub Actions で `npm run build` → `dist/` を GitHub Pages に自動デプロイ（main への push で公開）
- [x] 配布した HTTPS URL を実機で確認（ホーム画面追加・オフライン起動・写真選択・データ保存）
- [x] 知り合い向けに「インストール手順（共有 → ホーム画面に追加）」の簡単な案内を用意
- [x] アクセス制限は「URL 共有のみ」（パスコード等は設けない。必要になれば後で検討）

**メモ:** GitHub Pages はサブパス（`https://<user>.github.io/<repo>/`）配信になるため、`base` 設定を忘れると真っ白画面になりやすい。SPA のためルーティングは現状ハッシュ/単一ページ構成で問題ないが、将来 URL ルーティングを足す場合は 404 フォールバックに注意する。

### Phase 2 — Google Drive バックアップ（機種変更でもデータを失わない）
ゴール: iPhone を買い替えても、新しい端末で自分の Google アカウントにログインすれば全データが復元される。
本アプリは単独ユーザー利用のため、リアルタイム同期ではなく「**自分の Google Drive へバックアップ → 別端末で復元**」というモデルにする。
ローカル(IndexedDB)を常用の保存先とし、Google Drive はバックアップ/復元先として扱う。

- [x] Google Cloud でプロジェクト作成 → OAuth クライアント ID 発行（`drive.appdata` または `drive.file` スコープ）
- [x] Google Identity Services (GIS) でブラウザ上のログイン（OAuth）を実装
- [x] リポジトリ層に Google Drive アダプタを追加（Phase 1 の抽象化を利用）
- [x] 全データを 1 つの JSON（pets / diary / expenses）にまとめて Drive にアップロード/ダウンロード
- [x] 写真は圧縮済み Blob を保存（実装方式: 写真も含め全データを1つの `MofuLink-backup.json` にbase64で同梱し、Drive に「見えるファイル」として保存。スコープは `drive.file`）
- [x] 「今すぐバックアップ」ボタンと「Drive から復元」ボタンを用意
- [ ] 任意: アプリ起動時や保存時に自動バックアップ（頻度は控えめに）※未実装（手動バックアップのみ）

**メモ:** Google Drive 連携は Supabase より初期設定（Google Cloud 側の OAuth 設定）が一手間多い。
ここは Claude Code に「Google Cloud 側で何を設定すればよいか」を手順込みで案内させること。

## 5. 決定事項・残課題

**決定済み:**
- アプリ名: **MofuLink（モフリンク）**
- Phase 2 のクラウド先: **Google Drive**（自分の Drive にバックアップ/復元するモデル）
- 利用は iPhone のみ（PWA をホーム画面に追加）
- **配布: GitHub Pages でホスティングし、HTTPS の URL を知り合いにのみ共有する**
- **公開範囲: 一般公開しない（知り合いのみ）。アクセス制限は「URL 共有のみ」方式**（パスコード等は設けない）

**残課題（実装しながら決めてよい）:**
- 家族共有の範囲（Phase 3 でどこまでやるか）
- 写真の Drive 保存先を appDataFolder（非表示）にするか、見える「MofuLink」フォルダにするか
- 配布リポジトリ名（＝ GitHub Pages のサブパス）をどうするか（`base` 設定に影響）
